查看文件的两个版本之间的差异如何帮助你了解引入的错误？
	版本差异可以明确显示文件被修改的部分,这缩小了查找引入的错误的范围.
   
从长远来看，能轻松访问文件的整个历史记录将如何提高编程者的效率？
	当出现需要回退到某个版本的情况时候可以不用重新编写和修改文件.
	而且能够访问文件的整个历史可以方便对比版本间的差异,便于优化和错误修正.
	
你认为，手动选择何时创建提交（像在 Git 中做的那样）与自动保存版本（如 Google Docs 所做的那样）各有何优缺点？
	手动提交可以自主控制提交的更改量,使版本逻辑更清晰,避免提交量大之后产生混乱.
	手动提交相对于自动提交缺点是比较繁琐,没有自动提交稳定.
	自动提交可以稳定便捷地提交更改,但是会造成提交量过大和版本之间逻辑混乱,对版本
	间的控制不利.

如何使用命令 git log 和 git diff 来查看文件的历史记录？
	git log 打印显示文件的历史记录ID和信息,git diff 对比显示历史记录文件中的修改部分.
	
使用版本控制，如何让你更有信心进行有可能会造成破坏的更改？
	版本控制可以恢复到之前提交的任意版本,并且便于查找出错的版本.可以放心进行大范围的更改,
	尝试破坏性的更改而无需担心造成不可逆的结果.
